YZANE-PDF-EXPORT-GUIDE

PDF Export Guide - yzane Markdown Exporter

Overview

This guide explains how to export all markdown files in the AI Standards KB to PDF using the yzane.markdown-pdf VS Code extension.

Quick Start

Prerequisites

  1. VS Code - Download

  2. yzane.markdown-pdf Extension:

    code --install-extension yzane.markdown-pdf

Three Ways to Export

Option 1: VS Code GUI (Easiest)

  1. Open the project folder in VS Code:

    code "c:\Users\plachy\Documents\Dev\AI\_KB\ai-standards-kb"

  2. Open any .md file (or use Explorer to select files)

  3. Export options:

    • Single file: Press Ctrl+Shift+P → Type markdown-pdf: Export (document) → Enter
    • All files: Press Ctrl+Shift+P → Type markdown-pdf: Export (all files in workspace) → Enter

  4. PDFs appear in pdf/ folder with same filename

Option 2: Using PowerShell Script (Automated)

Run this script to export all markdown files:

cd "c:\Users\plachy\Documents\Dev\AI\_KB\ai-standards-kb"
.\export-yzane-quick.ps1

This script: - ✓ Verifies VS Code installation - ✓ Identifies all .md files - ✓ Opens each file and triggers export - ✓ Moves PDFs to pdf/ directory - ✓ Provides summary report

Option 3: Advanced Script with Fallbacks

If Pandoc or Node.js markdown-pdf is installed, this script provides multiple export methods:

.\export-to-pdf-yzane.ps1

Falls back to: - ✓ Pandoc (if installed) - ✓ Node.js markdown-pdf package (if installed)

yzane.markdown-pdf Features

Configuration

Create or edit .vscode/settings.json in your project:

{
  "markdown-pdf.type": "pdf",
  "markdown-pdf.displayHeaderFooter": true,
  "markdown-pdf.headerTemplate": "<div style='width: 100%; text-align: center; font-size: 10px;'><span class='title'></span></div>",
  "markdown-pdf.footerTemplate": "<div style='width: 100%; text-align: center; font-size: 10px;'>Page <span class='pageNumber'></span> of <span class='totalPages'></span></div>",
  "markdown-pdf.margin.top": "1cm",
  "markdown-pdf.margin.right": "1cm",
  "markdown-pdf.margin.bottom": "1cm",
  "markdown-pdf.margin.left": "1cm",
  "markdown-pdf.styles": [],
  "markdown-pdf.highlight": true,
  "markdown-pdf.breaks": false,
  "markdown-pdf.theme": "default",
  "markdown-pdf.scale": 1,
  "markdown-pdf.codeBlockTheme": "github"
}

Common Options

Setting Value Effect
displayHeaderFooter true Show page numbers
margin.top 1cm Top margin
theme default Light theme
theme dark Dark theme
codeBlockTheme github Code syntax highlighting
scale 1 100% zoom (1.0 = 100%, 0.8 = 80%)
breaks false Keep newlines as rendered

Supported Export Formats

  • ✓ PDF (default)
  • ✓ HTML (with "markdown-pdf.type": "html")
  • ✓ JPEG (with "markdown-pdf.type": "jpeg")
  • ✓ PNG (with "markdown-pdf.type": "png")

Troubleshooting

Problem: “markdown-pdf: Export” command not found

Solution: Install the extension:

code --install-extension yzane.markdown-pdf

Problem: PDFs not created in expected location

Solution: Check VS Code settings: 1. Open VS Code settings (Ctrl+,) 2. Search for “markdown-pdf” 3. Verify all settings are correct 4. Restart VS Code

Problem: Mathematical formulas not rendering correctly

Solution: This is a known limitation with some PDF renderers. Try: 1. Use Pandoc instead (better math support): powershell choco install pandoc # or download from https://pandoc.org

  1. Or configure yzane to use a different rendering engine

Problem: Code syntax highlighting is poor

Solution: Update codeBlockTheme in settings:

{
  "markdown-pdf.codeBlockTheme": "monokai"  // or "atom", "github", "solarized-dark"
}

Problem: Large files timeout during conversion

Solution: Increase the timeout in VS Code settings or split into smaller files

Advanced Usage

Batch Export Script

The export-yzane-quick.ps1 script handles batch operations:

# Export all files
.\export-yzane-quick.ps1

# Export with custom output directory
.\export-yzane-quick.ps1 -ProjectRoot "C:\path\to\project"

Using Pandoc as Alternative

If yzane doesn’t meet your needs, use Pandoc:

# Install Pandoc
choco install pandoc

# Export single file
pandoc standards/01-Neural-Networks-Fundamentals.md -o pdf/01-Neural-Networks-Fundamentals.pdf

# Export all with table of contents
Get-ChildItem standards/*.md | ForEach-Object {
  $basename = [System.IO.Path]::GetFileNameWithoutExtension($_)
  pandoc $_.FullName -o "pdf/$basename.pdf" --toc --number-sections
}

Using Node.js markdown-pdf

# Install globally
npm install -g markdown-pdf

# Export single file
markdown-pdf standards/01-Neural-Networks-Fundamentals.md

# Export all
Get-ChildItem standards/*.md | ForEach-Object {
  markdown-pdf $_.FullName
}

Quality Tips

  1. Review PDFs after export - Check for:
    • ✓ All pages rendered correctly
    • ✓ Code blocks formatted properly
    • ✓ Mathematical formulas visible
    • ✓ Mermaid diagrams rendered
    • ✓ Images and links working
  2. Optimize for distribution:
    • Consider PDF compression: Compress-PDF.ps1 script
    • Add metadata (title, author, subject)
    • Include table of contents
  3. Color mode:
    • For screen viewing: Use color theme
    • For printing: Use B&W theme to save ink
