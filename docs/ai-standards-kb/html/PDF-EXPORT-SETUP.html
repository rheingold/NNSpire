PDF-EXPORT-SETUP

PDF Export Scripts - Complete Setup

📦 What’s New

I’ve created 3 comprehensive PowerShell scripts + 2 detailed guides to help you export all markdown files to PDF using the yzane markdown exporter.

🎯 Quick Start (TL;DR)

# 1. Install the extension (one time only)
code --install-extension yzane.markdown-pdf

# 2. Run the batch export
cd "c:\Users\plachy\Documents\Dev\AI\_KB\ai-standards-kb"
.\export-batch-yzane.ps1

# 3. Check pdf/ folder for results ✓

📜 Available Scripts

1. export-batch-yzane.ps1 ⭐ (RECOMMENDED)

Best for: Automated batch export with progress tracking

Features: - ✓ Discovers all markdown files automatically - ✓ Shows real-time progress for each file - ✓ Handles errors gracefully - ✓ Creates detailed log file - ✓ Verifies VS Code & extension installation - ✓ Organized output with color-coded status

Usage:

.\export-batch-yzane.ps1
.\export-batch-yzane.ps1 -OpenVSCode  # Opens VS Code after export

Output: Creates export.log with full details

2. export-to-pdf-yzane.ps1

Best for: Fallback with multiple export methods

Features: - ✓ Tries VS Code + yzane first - ✓ Falls back to Pandoc if installed - ✓ Falls back to Node.js markdown-pdf if installed - ✓ Beautiful formatted output - ✓ Installation help for missing tools

Usage:

.\export-to-pdf-yzane.ps1

3. export-yzane-quick.ps1

Best for: Quick export with VS Code GUI automation

Features: - ✓ Fast startup - ✓ Minimal dependencies - ✓ Clear manual instructions - ✓ Works with VS Code workspace

Usage:

.\export-yzane-quick.ps1

🔧 Installation & Setup

Step 1: Install yzane Extension

code --install-extension yzane.markdown-pdf

Step 2: Verify Installation

code --list-extensions | Select-String "yzane"
# Output should show: yzane.markdown-pdf

Step 3: Configure (Optional)

Create .vscode/settings.json in project root:

{
  "markdown-pdf.displayHeaderFooter": true,
  "markdown-pdf.margin.top": "1cm",
  "markdown-pdf.margin.bottom": "1cm",
  "markdown-pdf.codeBlockTheme": "github",
  "markdown-pdf.theme": "default"
}

⚙️ Configuration

Common Settings

Setting Value Effect
Theme default Light background
Theme dark Dark background
Code Theme github Nice syntax highlighting
Display Header/Footer true Add page numbers
Scale 1 100% zoom (use 0.8 for 80%)

Edit Settings

Create .vscode/settings.json:

{
  "markdown-pdf.type": "pdf",
  "markdown-pdf.displayHeaderFooter": true,
  "markdown-pdf.headerTemplate": "<div style='width: 100%; text-align: center;'></div>",
  "markdown-pdf.footerTemplate": "<div style='width: 100%; text-align: center; font-size: 10px;'>Page <span class='pageNumber'></span>/<span class='totalPages'></span></div>",
  "markdown-pdf.theme": "default",
  "markdown-pdf.codeBlockTheme": "github",
  "markdown-pdf.scale": 1,
  "markdown-pdf.margin": {
    "top": "1cm",
    "right": "1cm",
    "bottom": "1cm",
    "left": "1cm"
  }
}

🐛 Troubleshooting

Problem: “Command not found”

# Make sure you're in the correct directory
cd "c:\Users\plachy\Documents\Dev\AI\_KB\ai-standards-kb"

# Check if script exists
ls export*.ps1

Problem: “Extension not found”

# Install it
code --install-extension yzane.markdown-pdf

# Verify installation
code --list-extensions | Select-String "yzane"

Problem: “No PDFs created”

# Try with logs
.\export-batch-yzane.ps1
# Check export.log for details

# Or use manual method:
# 1. code .
# 2. Ctrl+Shift+P → markdown-pdf: Export (all files in workspace)

Problem: “Math formulas not rendering”

Use Pandoc instead (better math support):

# Install
choco install pandoc

# Then run fallback script
.\export-to-pdf-yzane.ps1
# It will detect Pandoc and use it

📈 Performance Tips

  1. Batch Processing: Export all files at once (faster than individual exports)
  2. Close Unnecessary Files: Reduces VS Code memory usage
  3. Disable Extensions: Temporarily disable other extensions if export is slow
  4. Check Disk Space: Ensure ~200MB free space for all PDFs
